## 1. 개요

S3 Spooler는 클라이언트 프로세스에게 **파일 시스템 인터페이스**만 제공한다.  
별도 IPC, SDK, 네트워크 연결 없이 지정된 스풀 디렉토리에 파일을 쓰는 것만으로 S3 업로드를 요청할 수 있다.

```
[클라이언트 프로세스]
    1. 파일 준비 (임시 경로에 생성)
    2. 스풀 파일명 생성 (encode)
    3. atomic rename → 스풀 디렉토리
         ↓
[S3 Spooler 자동 처리]
    4. 파일 감지
    5. Stream Manager 전송
    6. 파일 삭제
```

## 2. 스풀 파일명 규칙

### 2.1 형식

```
{stream_id}__{s3_key_encoded}
```

| 구성 요소 | 규칙 | 예시 |
|-----------|------|------|
| `stream_id` | regex: `^[a-z][a-z0-9\-_]*$` (최대 64자). 소문자로 시작, 소문자·숫자·하이픈·언더스코어만 포함. 형식: `{process_name}` (고정) | `telemetry`, `device-001`, `camera-front` |
| `s3_key_encoded` | S3 업로드 경로(키)의 슬래시를 느낌표(!)로 치환 | `data!device-1!reading.json` |
| 구분자 | `__` (이중 언더스코어) | |

### 2.1a stream_id 명명 규칙

**형식**: `{process_name}` (고정 스트림)

- `process_name`: 프로세스/기기 식별자 (영문 소문자, 숫자, 하이픈 혼용)
- **sequence는 S3 키에서 구분**: 동적 구분이 필요하면 파일명의 `s3_key_encoded` 부분에서 경로로 처리

| 항목 | 규칙 |
|------|------|
| **허용 문자** | 소문자 영문자, 숫자, 하이픈(`-`), 언더스코어(`_`) |
| **첫 글자** | 소문자 영문자(`a-z`) |
| **금지** | 대문자, `__`, 공백, 특수문자, 숫자 시작 |
| **길이** | 1~64자 |
| **권장 컨벤션** | kebab-case (예: `telemetry`, `device-001`, `camera-front`) |

**유효성 검증**: stream_id 규칙을 위반하면 `ValueError` 발생하며, 파일이 스풀 디렉토리에 쓰여지지 않음.

**유효한 예 (고정 스트림)**:
```
telemetry         # 프로세스명 단독
device-001        # 프로세스명 + 식별번호 (하나의 고정 스트림)
camera-front      # 프로세스명 + 위치 (하나의 고정 스트림)
sensor-data       # 언더스코어 사용 가능 (권장하지 않음)
```

**동적 구분은 S3 키에서 처리**:
```
# 잘못된 방법 (이전 설계)
stream_id: sensor-001, sensor-002 (여러 스트림)
s3_key: data/reading.json

# 올바른 방법 (신규 설계)
stream_id: sensor (하나의 고정 스트림)
s3_key: data/device-001/reading.json, data/device-002/reading.json
```

**무효한 예**:
```
Telemetry         # 대문자
1-sensor          # 숫자 시작
my__stream        # 이중 언더스코어 (구분자와 충돌)
```

### 2.2 인코딩 도구 (세 가지 방법)

클라이언트 언어와 환경에 따라 하나를 선택한다.

---

#### 방법 A: CLI 도구 (모든 언어·환경에서 사용 가능)

스풀러가 설치된 환경에서 subprocess로 호출한다.

```bash
# 인코딩
python3 -m spooler encode <stream_id> <s3_key>

# 예시
SPOOL_NAME=$(python3 -m spooler encode "telemetry" "data/device-1/reading.json")
echo $SPOOL_NAME
# → telemetry__data!device-1!reading.json

# 파일 이동 (atomic rename)
cp /tmp/sensor_data.json /tmp/sensor_data_tmp.json
mv /tmp/sensor_data_tmp.json /var/spool/s3-spooler/$SPOOL_NAME
```

---

#### 방법 B: Python 라이브러리

```python
from pathlib import Path
from spooler.filename_codec import encode

# 스풀 파일명 생성
spool_name = encode(
    stream_id="telemetry",
    s3_key="data/device-1/reading.json",
)
# → "telemetry__data!device-1!reading.json"

# Atomic rename으로 스풀 디렉토리에 이동
src = Path("/tmp/sensor_data.json")
dst = Path("/var/spool/s3-spooler") / spool_name

# 임시 파일을 같은 파일시스템으로 먼저 이동 후 rename
import shutil, os
tmp = dst.parent / f".tmp_{spool_name}"
shutil.copy2(src, tmp)     # 원본 → 스풀 디렉토리 내 임시 파일
os.rename(tmp, dst)        # atomic rename (같은 파일시스템)
```

---

#### 방법 C: Shell 스크립트 (bash/sh, 외부 의존성 없음)

```bash
#!/bin/bash
# encode_spool_name.sh
# 사용법: encode_spool_name.sh <stream_id> <s3_key>

STREAM_ID="$1"
S3_KEY="$2"
ORIGINAL_FILENAME=$(basename "$S3_KEY")

# 슬래시를 느낌표로 치환 (간단한 인코딩)
ENCODED=$(printf '%s' "$S3_KEY" | tr '/' '!')

echo "${STREAM_ID}__${ENCODED}"
```

```bash
# 사용 예
SPOOL_NAME=$(./encode_spool_name.sh "telemetry" "data/device-1/reading.json")
cp /tmp/sensor.json /tmp/.sensor_tmp.json
mv /tmp/.sensor_tmp.json /var/spool/s3-spooler/$SPOOL_NAME
```

### 2.3 stream_id 와 Stream Manager 스트림

`stream_id`는 **Stream Manager 스트림 이름으로 직접 사용**된다.

1. 파일명에서 `stream_id` 추출
2. `stream_id`를 Stream Manager 스트림 이름으로 그대로 사용
3. 해당 이름의 스트림이 Stream Manager에 존재하면 전송 성공
4. 스트림이 없으면 전송 실패 → 파일은 보존되어 재시도 가능

> 사용할 stream_id 목록과 대응 스트림 생성은 관리자 책임이다 ([08-S3-스트림-관리-가이드](08-S3-스트림-관리-가이드.md) 참고).

### 2.4 s3_key 규칙

| 조건 | 내용 |
|------|------|
| 형식 | `{prefix_path}/{filename}` (슬래시로 경로 구분) |
| 파일명 | 비어있으면 안 됨 (s3_key가 슬래시로 끝나면 오류) |
| 인코딩 | 슬래시(`/`)를 느낌표(`!`)로 치환 (UTF-8 안전 — 한글·공백 포함 가능) |
| 최종 S3 키 | `s3_key` 값이 **그대로** 실제 S3 오브젝트 키가 됨 (Pattern 2, per-file 제어) |

## 3. 파일 쓰기 절차 (경쟁 조건 방지)

> **반드시 준수**: 스풀 디렉토리에 직접 쓰면 쓰기 중인 파일이 감지될 수 있다.

**권장 절차**:

```
1. 임시 파일 생성 (/tmp 또는 스풀 디렉토리 내 .tmp_ 접두사)
2. 최종 데이터 쓰기 (flush/close 완료)
3. encode()로 스풀 파일명 계산
4. os.rename() / mv 로 atomic하게 스풀 디렉토리로 이동
```

**같은 파일시스템 내에서만 atomic rename이 보장됨**:
- 임시 파일이 스풀 디렉토리와 **같은 파티션/마운트포인트**에 있어야 한다
- 다른 파티션이면 `shutil.copy2()` 후 `os.rename()` 사용 (위 예시 참고)

## 3.1 파일 안정성 검증 동작 (하이브리드 방식)

S3 Spooler는 **하이브리드 안정성 검증**을 통해 파일 쓰기 완성도를 확인한다:

### 일반적인 경우 (빠른 쓰기)
```
파일 생성 감지 → 시간 기반 대기 (~100ms) → 즉시 전송
```

### 느린 쓰기/네트워크 스토리지
```
파일 생성 감지 → 시간 기반 대기 → 크기 추적 → 안정화 후 전송
├── 200ms 간격으로 파일 크기 체크
├── 3회 연속 크기 불변 시 안정 판정  
└── 최대 10초 대기 후 타임아웃
```

### 클라이언트 측 최적화

**권장**: atomic rename으로 경쟁 조건 완전 제거
```bash
# 좋은 예: 같은 파일시스템에서 atomic rename
cp source.json /tmp/temp_file.json
mv /tmp/temp_file.json /var/spool/s3-spooler/target_name

# 나쁜 예: 스풀 디렉토리에 직접 쓰기 (쓰기 중 감지 가능)
cp source.json /var/spool/s3-spooler/target_name  # 비권장
```

**파일시스템 경계 처리**:
```python
# 다른 파티션 간 이동 시
import shutil, os
from pathlib import Path

src = Path("/different/partition/source.json")
spool_dir = Path("/var/spool/s3-spooler")
tmp = spool_dir / f".tmp_{target_name}"
final = spool_dir / target_name

shutil.copy2(src, tmp)    # 데이터 복사
os.rename(tmp, final)     # 같은 파일시스템에서 atomic rename
```

### 안정성 튜닝 (INI 설정)

느린 스토리지 환경에서는 안정성 파라미터를 조정할 수 있다:

```ini
[stability]
# 시간 기반 대기 증가 (기본 0.1초)
file_stability_wait = 0.5

# 크기 체크 간격 증가 (기본 0.2초)  
stability_check_interval = 1.0

# 연속 불변 확인 횟수 증가 (기본 3회)
stability_check_count = 5

# 최대 대기 시간 증가 (기본 10초)
max_stability_wait = 30.0
```

## 5. 파일 크기 제한

| 제한 | 값 | 이유 |
|------|-----|------|
| 단일 파일 최대 크기 | 실질적 제한 없음 | Pattern 2: Stream Manager가 S3 멀티파트 자동 처리 |
| 스풀 전체 한도 | 900 MB (기본) | 전용 1GB 파티션 기준 (FR-05) |

클라이언트 측 분할은 불필요하다. 임의 크기 파일을 그대로 스풀 디렉토리에 두면 단일 S3 오브젝트로 업로드된다.

---

## 6. 오류 처리

| 상황 | 클라이언트 동작 |
|------|----------------|
| 스풀 디렉토리가 없음 | Spooler가 자동 생성. 없으면 쓰기 전 `mkdir -p` |
| 파일 전송 실패 | Spooler가 파일을 삭제하지 않고 보존. 보존 기간 내 재시도 가능 |
| 파일이 삭제되지 않음 (장시간) | 용량 초과 또는 보존 기간 만료로 Spooler가 삭제 → 데이터 손실 |
| stream_id가 잘못됨 | Spooler가 전송 실패 로그 기록. 스트림 이름 확인 필요 |

## 7. 예제: 센서 데이터 주기적 전송 (Python)

```python
import time
import json
import os
import shutil
from pathlib import Path
from spooler.filename_codec import encode

SPOOL_DIR = Path("/var/spool/s3-spooler")
STREAM_ID = "sensor-telemetry"
DEVICE_ID = "device-001"

def send_sensor_reading(temperature: float, humidity: float) -> None:
    payload = {
        "device_id": DEVICE_ID,
        "timestamp": time.time(),
        "temperature": temperature,
        "humidity": humidity,
    }
    filename = f"reading_{int(time.time())}.json"
    s3_key = f"sensors/{DEVICE_ID}/{filename}"
    spool_name = encode(STREAM_ID, s3_key)

    # 임시 파일에 쓰기
    tmp = SPOOL_DIR / f".tmp_{spool_name}"
    tmp.write_text(json.dumps(payload))

    # Atomic rename
    os.rename(tmp, SPOOL_DIR / spool_name)
    print(f"Queued: {s3_key}")

# 10초마다 전송
while True:
    send_sensor_reading(temperature=25.3, humidity=60.1)
    time.sleep(10)
```
