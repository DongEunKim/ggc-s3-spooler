## 1. 시스템 환경

### 1.1 운영 환경 — TGU (Telematic Gateway Unit)

이 컴포넌트의 실제 운영 환경은 **건설기계에 탑재된 TGU**이다.  
아래 항목 중 ⚠️ 표시된 항목은 실제 TGU에서 확인이 필요하다. ([research/06](research/06-tgu-platform-constraints.md) 참고)

| 항목 | 현재 알려진 사양 | 확인 필요 여부 |
|------|----------------|----------------|
| 디바이스 형태 | 건설기계 탑재 TGU | — |
| OS | Linux | ⚠️ 배포판·버전 미확정 |
| CPU 아키텍처 | ARM (추정) | ⚠️ aarch64 vs armv7l 미확정 |
| Python 버전 | 3.11 (목표) | ⚠️ 실제 설치 버전 미확인 |
| 총 저장공간 | 1 GB | — ([research/03](research/03-spool-sizing-rationale.md)) |
| 네트워크 | AWS IoT Core 연결 가능, PyPI 차단 | — |
| pip3 | 설치됨 (PyPI 접근만 차단) | ⚠️ pip 버전 미확인 |
| stream-manager SDK | **미설치** | — |
| AWS Greengrass | v2.x | — |
| Stream Manager 컴포넌트 | ≥ 2.1.0 | — |

### 1.2 런타임 의존성 및 배포 방식

PyPI 접근이 불가하므로 모든 런타임 의존성은 **아티팩트 zip에 포함**되어야 한다.

| 패키지 | 버전 | 용도 | wheel 타입 |
|--------|------|------|------------|
| `watchdog` | ≥ 4.0.0 | 파일시스템 감시 | `py3-none-manylinux2014_aarch64` (pure Python on Linux) |
| `stream-manager` | ≥ 1.1.1 | Stream Manager IPC SDK | `py2.py3-none-any` (pure Python) |
| `cbor2` | ≥ 5.4.6, < 6.0 | stream-manager 직접 의존성 | `cp311-cp311-manylinux_2_17_aarch64` (C ext, Python 버전 의존) |

> **cbor2 주의**: `cp311` wheel은 Python 3.11에서만 동작한다.  
> TGU Python 버전이 다른 경우 해당 버전의 wheel로 재빌드 필요.

## 2. 성능 사양

| 항목 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 파일 감지 지연 | < 1초 | 파일 생성 → Queue push 시간 |
| 전송 처리량 | ≥ 10 파일/초 (소형 파일 기준) | 단위 테스트 벤치마크 |
| 메모리 사용량 | < 128MB | RSS 기준, 1000 파일 동시 처리 |
| 정리 사이클 오버헤드 | < 100ms | 10,000 파일 디렉토리 기준 |

### 2.1 하이브리드 안정성 검증 성능 특성

| 시나리오 | 예상 지연시간 | 비고 |
|----------|---------------|------|
| **일반적 파일** (빠른 쓰기) | ~100ms | 시간 기반 필터링만으로 처리 |
| **느린 쓰기** (네트워크 스토리지) | 200ms~10s | 크기 기반 검증으로 완성 대기 |
| **대용량 파일** (>100MB) | 500ms~10s | 멀티파트 쓰기 완성도 추적 |
| **타임아웃** (max_stability_wait) | 10s | 강제 진행, 전송 실패 가능성 있음 |

**튜닝 파라미터**:
- `file_stability_wait`: 시간 기반 대기 (기본 0.1초)
- `stability_check_interval`: 크기 체크 간격 (기본 0.2초)  
- `stability_check_count`: 연속 불변 확인 횟수 (기본 3회)
- `max_stability_wait`: 최대 대기 시간 (기본 10초)

## 5. 설정 파라미터 상세

### 5.1 설정 시스템 (CAN Blackbox 스타일 INI 지원)

**설정 우선순위**: CLI Arguments > INI File > Default Values

```bash
# CLI로 설정 오버라이드
python -m spooler --spool-dir /custom/path --config custom.ini --log-level DEBUG

# INI 파일 예시 (spooler.ini)
[spooler]
spool_dir = /var/spool/s3-spooler/spool
log_level = INFO

[stability]
file_stability_wait = 0.1
max_stability_wait = 10.0
```

### 5.2 전체 파라미터 목록

| 파라미터 | INI 섹션 | 타입 | 범위 | 기본값 | 설명 |
|----------|----------|------|------|--------|------|
| `spool_dir` | [spooler] | path | 유효한 경로 | `/var/spool/s3-spooler/spool` | 스풀 디렉토리 |
| `log_level` | [spooler] | enum | DEBUG/INFO/WARNING/ERROR | `INFO` | 로그 레벨 |
| `max_spool_size_mb` | [cleanup] | int | 1~10240 | 900 | 최대 스풀 크기 (MB) |
| `file_retention_hours` | [cleanup] | int | 1~8760 | 24 | 파일 최대 보존 시간 |
| `poll_interval_seconds` | [cleanup] | int | 1~3600 | 5 | 정리 사이클 주기 |
| `stream_manager_host` | [stream_manager] | string | hostname | `localhost` | Stream Manager 주소 |
| `stream_manager_port` | [stream_manager] | int | 1~65535 | 8088 | Stream Manager 포트 |
| **`file_stability_wait`** | [stability] | float | 0.0~60.0 | 0.1 | **시간 기반 안정성 대기 (초)** |
| **`stability_check_interval`** | [stability] | float | 0.1~10.0 | 0.2 | **크기 체크 간격 (초)** |
| **`stability_check_count`** | [stability] | int | 1~100 | 3 | **연속 불변 확인 횟수** |
| **`max_stability_wait`** | [stability] | float | 1.0~300.0 | 10.0 | **최대 안정성 대기 시간 (초)** |

### 5.3 INI 파일 전체 예시

```ini
[spooler]
spool_dir = /var/spool/s3-spooler/spool
log_level = INFO

[stream_manager]
host = localhost
port = 8088

[cleanup]
max_spool_size_mb = 900
file_retention_hours = 24
poll_interval_seconds = 5

[stability]
file_stability_wait = 0.1
stability_check_interval = 0.2
stability_check_count = 3
max_stability_wait = 10.0
```

## 6. 오류 처리 정책

| 오류 유형 | 처리 방법 | 재시도 |
|-----------|-----------|--------|
| Stream Manager 연결 실패 | ERROR 로그, 다음 파일 처리 계속 | 없음 (파일 보존) |
| 파일 읽기 실패 | ERROR 로그, 해당 파일 건너뜀 | 없음 |
| 파일 삭제 실패 | ERROR 로그, 처리 계속 | 없음 |
| 파일명 디코딩 실패 | WARNING 로그, 해당 파일 건너뜀 | 없음 |
| 정리 사이클 예외 | ERROR 로그, 다음 사이클 계속 | 자동 (다음 주기) |

> **개선 검토 (A-01)**: 전송 실패 파일의 지수 백오프 재시도 큐 도입을 검토한다.
