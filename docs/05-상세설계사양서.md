## 1. filename_codec 모듈

### 1.1 encode()

```
encode(stream_id: str, s3_key: str) -> str
```

**알고리즘**:
1. `stream_id` 검증 (순서대로):
   - 빈 문자열 검사 → ValueError
   - 길이 검사 (> 64자) → ValueError
   - regex `^[a-z][a-z0-9\-_]*$` 검사 → ValueError
   - `__` 포함 여부 검사 (명시적 에러) → ValueError
2. `s3_key`에서 `Path(s3_key).name`으로 `original_name` 추출 (검증용)
3. `s3_key`의 슬래시(/)를 느낌표(!)로 치환하여 `s3_key_encoded` 생성
4. `{stream_id}__{s3_key_encoded}` 반환

**엣지 케이스**:
- `s3_key`가 슬래시로 끝나면 `original_name`이 빈 문자열 → ValueError
- 한글, 특수문자 포함 s3_key는 슬래시 치환 후 그대로 유지 (UTF-8 안전)

### 1.2 decode()

```
decode(spool_filename: str) -> SpoolFileMeta
```

**알고리즘**:
1. `split("__", 1)`로 2개 파트 분리
2. 파트가 2개 미만이면 ValueError
3. `s3_key_encoded`의 느낌표(!)를 슬래시(/)로 복원하여 `s3_key` 생성
4. `Path(s3_key).name`으로 `original_name` 추출
5. `SpoolFileMeta` dataclass 반환

## 2. SpoolWatcher 모듈

### 2.1 파일 감지 → 전송 파이프라인

```
watchdog Observer (별도 스레드)
  ↓ FileCreatedEvent
SpoolEventHandler.on_created()
  ↓ loop.call_soon_threadsafe(queue.put_nowait, path)
asyncio Queue[Path]
  ↓ await queue.get()
SpoolWatcher._transfer(path)
  ↓ await run_in_executor(None, client.append_file, ...)
StreamManagerClient.append_file()
  ↓ 성공 시
path.unlink(missing_ok=True)
```

### 2.2 기동 시 잔류 파일 처리

`_drain_existing()`은 `process_loop()` 시작 시 한 번 실행된다.  
`watchdog.Observer`보다 먼저 실행되므로 Observer 시작 전 도착한 파일도 처리된다.  
`spool_dir.iterdir()`을 mtime 오름차순 정렬하여 처리 순서를 보장한다.

### 2.3 경쟁 조건 분석

| 시나리오 | 대응 |
|----------|------|
| 파일이 쓰여지는 중 감지 | 클라이언트가 atomic rename 사용 권고 (인터페이스 설계서 §1.2) |
| 전송 중 cleaner가 삭제 | `path.unlink(missing_ok=True)` + 읽기 실패 시 로그 후 건너뜀 |
| 동일 파일 이중 감지 | `_transfer()` 시작 시 `path.exists()` 확인 |

## 3. SpoolCleaner 모듈

### 3.1 _evict_expired()

```python
cutoff = time.time() - retention_hours * 3600
for path in iter_spool_files():
    if path.stat().st_mtime < cutoff:
        path.unlink(missing_ok=True)
```

**주의**: `OSError`(파일이 전송 중 삭제 등)는 로그 후 계속 진행.

### 3.2 _evict_over_quota()

```python
files = sorted(iter_spool_files(), key=lambda p: p.stat().st_mtime)
total = sum(f.stat().st_size for f in files)
for path in files:
    if total <= max_bytes:
        break
    total -= path.stat().st_size
    path.unlink(missing_ok=True)
```

**주의**: `stat()` 호출 시점과 `unlink()` 사이에 파일이 사라질 수 있음. `missing_ok=True`와 `OSError` 처리로 대응.

## 4. StreamManagerClient 모듈

### 4.1 실제 클라이언트 (StreamManagerClient)

`stream_manager` 패키지는 Greengrass 런타임에서만 사용 가능하다.  
`connect()` 시 import하여 로컬 개발 환경에서 import 오류를 방지한다.

### 4.2 Mock 클라이언트 (MockStreamManagerClient)

테스트 시 `MockStreamManagerClient`로 교체:
```python
# conftest.py 예시
@pytest.fixture
def stream_client():
    return MockStreamManagerClient()
```

`sent` 리스트에 `(stream_name, s3_key, data)` 튜플을 누적하여 테스트 검증에 사용한다.

## 5. 설정 모듈 (config.py)

`SpoolerConfig`는 `dataclass`로 정의되어 있으며,  
`from_args(argparse.Namespace)` 클래스 메서드로 CLI 인수로부터 생성된다.

### 5.1 SpoolerConfig 필드

| 필드명 | 타입 | 기본값 | 설명 |
|--------|------|--------|------|
| `spool_dir` | Path | `/tmp/s3-spooler/spool` | 스풀 디렉토리 경로 |
| `max_spool_size_mb` | int | 900 | 최대 스풀 크기 (MB) — 전용 1GB 파티션 기준 |
| `file_retention_hours` | int | 24 | 파일 보존 기간 (시간) |
| `poll_interval_seconds` | int | 5 | 정리 사이클 주기 (초) |
| `stream_manager_host` | str | localhost | Stream Manager 호스트 |
| `stream_manager_port` | int | 8088 | Stream Manager 포트 |
| `log_level` | str | INFO | 로그 레벨 |
| `filename_separator` | str | `__` | 파일명 구분자 (내부 사용) |
| `registry_path` | Path | `{spool_dir}/../registry.json` | 스트림 → 버킷 레지스트리 파일 경로 (신규) |
| `status_stream_name` | str | `""` | A-07 상태 스트림 이름 (신규) |

### 5.2 계산된 속성

- `max_spool_size_bytes`: `max_spool_size_mb * 1024 * 1024` — 바이트 단위 최대 크기

향후 환경변수 오버라이드 지원 시 `pydantic-settings`로 마이그레이션을 검토한다.  
(`pydantic`은 이미 의존성에 포함됨)

## 6. 스트림 레지스트리 모듈 (stream_registry.py)

스트림명 → S3 버킷 매핑을 JSON 파일로 관리하는 **선택적 관리 도구**다.

**역할**: 
- 특정 stream_id를 특별한 S3 버킷으로 라우팅하는 설정 관리
- 관리자가 스트림과 버킷 관계를 추적하기 위한 도구
- **실제 전송 동작에는 직접적 영향 없음** (Stream Manager 스트림 존재 여부가 실제 결정 요소)

### 6.1 레지스트리 파일 위치

```
{spool_dir}/../registry.json
```

예: spool_dir = `/var/spool/s3-spooler/spool` → registry = `/var/spool/s3-spooler/registry.json`

이 위치는 watchdog observer의 감시 범위 외이므로 스풀 파일과 혼동되지 않는다.

### 6.2 파일 형식

```json
{
  "telemetry": "my-telemetry-bucket",
  "alarm": "my-alarm-bucket",
  "camera-front": "my-video-bucket"
}
```

### 6.3 StreamRegistry 클래스 메서드

| 메서드 | 시그니처 | 설명 |
|--------|---------|------|
| `load()` | `() → Dict[str, str]` | 레지스트리 파일 읽기. 파일 없으면 빈 dict 반환. |
| `get_bucket()` | `(stream_name: str) → str \| None` | 스트림명으로 버킷 조회. 미등록 시 None. |
| `add()` | `(stream_name: str, bucket: str) → None` | 스트림 추가/갱신. stream_name 검증 필수. |
| `remove()` | `(stream_name: str) → None` | 스트림 제거. 미등록 시 ValueError. |
| `list_all()` | `() → Dict[str, str]` | 전체 매핑 반환. |

### 6.4 stream_name 검증

`add()` 메서드는 stream_name을 `filename_codec.STREAM_ID_PATTERN`으로 검증한다.
- regex: `^[a-z][a-z0-9\-_]*$`
- 길이: 1~64자

### 6.5 원자적 쓰기 (Atomic writes)

파일 손상을 방지하기 위해 tmp → rename 패턴 사용:

```python
tmp_path = registry_path.with_suffix(".tmp")
# 1. tmp 파일에 쓰기
with open(tmp_path, "w") as f:
    json.dump(data, f)
# 2. 원자적 rename (같은 파일시스템)
tmp_path.replace(registry_path)
```

부모 디렉토리가 없으면 자동 생성된다.
