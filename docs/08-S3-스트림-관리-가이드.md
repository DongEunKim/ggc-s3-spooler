## 1. 개요

S3 Spooler 컴포넌트는 **이미 생성된 Stream Manager 스트림**을 사용한다.  
이 가이드는 스트림 생성, S3 버킷 설정, IAM 권한, S3 키 구조를 다룬다.

**관리자 책임**:
- **필수**: Stream Manager에 스트림 사전 생성 (클라이언트가 사용할 stream_id별)
- **선택적**: 특별한 S3 버킷 라우팅이 필요한 stream_id를 레지스트리에 등록
- **주의**: 클라이언트는 등록되지 않은 stream_id도 사용할 수 있으며, Stream Manager 스트림 존재 여부가 실제 전송 성공/실패를 결정

## 2. 스트림 관리

### 2.1 스트림 명명 규칙

| 조건 | 내용 |
|------|------|
| 허용 문자 | 영문자, 숫자, 하이픈(-), 언더스코어(_) |
| 금지 | `__` (이중 언더스코어) — 스풀 파일명 구분자와 충돌 |
| 길이 | 1~128자 |
| 예시 | `telemetry`, `device-events`, `camera_frames`, `alarm-data` |

### 2.2 스트림 생성 (Python SDK) — 올바른 코드

> ⚠️ **중요**: `S3ExportTaskExecutorConfig`에는 `s3_bucket`, `key_prefix` 파라미터가 없다.  
> 버킷과 S3 키는 스트림 수준이 아닌 **메시지마다** `S3ExportTaskDefinition`으로 지정한다.  
> (SDK 1.1.1 소스 직접 확인 완료 — 2026-06-01)

```python
from stream_manager import StreamManagerClient
from stream_manager.data import (
    MessageStreamDefinition,
    S3ExportTaskExecutorConfig,
    S3ExportTaskDefinition,
    ExportDefinition,
    StatusConfig,
    StatusLevel,
    StrategyOnFull,
    Persistence,
)
from stream_manager.util import Util

client = StreamManagerClient(host="localhost", port=8088)

# 1. (권장) 업로드 완료 상태를 수신할 상태 스트림 먼저 생성
client.create_message_stream(
    MessageStreamDefinition(name="telemetry-status-stream")
)

# 2. S3 Export 스트림 생성
client.create_message_stream(
    MessageStreamDefinition(
        name="telemetry",
        max_size=268_435_456,               # 스트림 버퍼 256MB (Greengrass 파티션)
        strategy_on_full=StrategyOnFull.OverwriteOldestData,
        persistence=Persistence.File,
        export_definition=ExportDefinition(
            s3_task_executor=[
                S3ExportTaskExecutorConfig(
                    identifier="S3Export-telemetry",
                    # ✅ 5MB 이상 파일은 자동으로 S3 멀티파트 업로드 — 64MB 한도 없음
                    size_threshold_for_multipart_upload_bytes=5_242_880,
                    status_config=StatusConfig(
                        status_level=StatusLevel.ERROR,
                        status_stream_name="telemetry-status-stream"
                    )
                )
            ]
        ),
    )
)

# 3. 파일 업로드 태스크 등록 (per-file 버킷/키 완전 제어)
task = S3ExportTaskDefinition(
    input_url="file:///var/spool/s3-spooler/telemetry__key__reading.json",
    bucket="my-iot-data-bucket",        # ← 버킷은 메시지마다 지정
    key="raw/telemetry/device-001/reading.json"  # ← S3 키도 메시지마다 지정
)
seq = client.append_message("telemetry", Util.validate_and_serialize_to_json_bytes(task))
# → Stream Manager가 파일을 읽어 S3에 업로드 (멀티파트 자동 적용)
# → 업로드 완료 후 telemetry-status-stream에 StatusMessage 기록
# ✅ JSON 직렬화 (올바른 방식) — cbor2.dumps() 사용 금지

client.close()
```

### 2.3 S3 키 결정 방식

S3 오브젝트 키는 `S3ExportTaskDefinition.key` 파라미터가 **직접 결정**한다.

```
S3 키 = S3ExportTaskDefinition.key 값 (스풀 파일명의 s3_key 디코딩 결과)
```

**예시**:
- 스풀 파일명: `telemetry__cmF3L2RldmljZS0wMDEvcmVhZGluZy5qc29u__reading.json`
- 디코딩: `s3_key = "raw/device-001/reading.json"`
- 업로드 버킷: `my-iot-data-bucket`
- **최종 S3 URI**: `s3://my-iot-data-bucket/raw/device-001/reading.json`

> **참고**: `key` 파라미터는 `!{timestamp:yyyy/MM/dd}` 형식의 Java DateTimeFormatter placeholder를 지원한다.  
> 예: `key = "data/!{timestamp:yyyy/MM/dd}/device-001/reading.json"`

### 2.4 스트림 목록 확인

```python
client = StreamManagerClient()
streams = client.list_streams()
for name in streams:
    print(name)
client.close()
```

또는 AWS IoT 콘솔 → Greengrass → Core devices → 해당 디바이스 → Stream Manager

### 2.5 스트림 삭제 (주의 필요)

```python
client.delete_message_stream(stream_name="telemetry")
```

> **경고**: 스트림 삭제 전 Spooler 컴포넌트를 중지해야 한다.  
> 실행 중 삭제 시 전송 실패 오류가 발생하며 스풀 파일이 보존 기간까지 남아있게 된다.

## 3. S3 버킷 설정

### 3.1 버킷 생성 체크리스트

```bash
# 버킷 생성
aws s3api create-bucket \
  --bucket my-iot-data-bucket \
  --region ap-northeast-2 \
  --create-bucket-configuration LocationConstraint=ap-northeast-2

# 버전 관리 활성화 (선택 — 덮어쓰기 보호)
aws s3api put-bucket-versioning \
  --bucket my-iot-data-bucket \
  --versioning-configuration Status=Enabled

# 암호화 설정 (권장)
aws s3api put-bucket-encryption \
  --bucket my-iot-data-bucket \
  --server-side-encryption-configuration '{
    "Rules": [{"ApplyServerSideEncryptionByDefault": {"SSEAlgorithm": "AES256"}}]
  }'

# 퍼블릭 액세스 차단 (보안)
aws s3api put-public-access-block \
  --bucket my-iot-data-bucket \
  --public-access-block-configuration \
    "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"
```

### 3.2 S3 키 구조 권장 설계

아래 구조를 권장한다 (파티션 쿼리 최적화):

```
{버킷}/
└── {서비스 이름}/
    └── {디바이스 ID 또는 그룹}/
        └── {날짜 YYYY/MM/DD}/   ← S3 Select / Athena 파티션
            └── {파일명}
```

**예시**:
```
my-iot-data-bucket/
├── telemetry/
│   ├── device-001/
│   │   └── 2026/05/31/
│   │       ├── reading_1748652000.json
│   │       └── reading_1748652010.json
│   └── device-002/
│       └── 2026/05/31/
│           └── reading_1748652005.json
└── alarms/
    └── site-A/
        └── 2026/05/31/
            └── alarm_critical_1748652100.json
```

클라이언트 측 s3_key 예시:
```python
from datetime import datetime
now = datetime.utcnow()
s3_key = f"device-001/{now.year}/{now.month:02}/{now.day:02}/reading_{int(time.time())}.json"
encode("telemetry", s3_key)
```

## 4. IAM 권한 설정

### 4.1 Greengrass Core Device IAM Role

Greengrass가 Stream Manager를 통해 S3에 쓰려면 Core Device의 IAM Role에 S3 쓰기 권한이 필요하다.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "S3SpoolerWrite",
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:PutObjectAcl"
      ],
      "Resource": "arn:aws:s3:::my-iot-data-bucket/*"
    }
  ]
}
```

> 최소 권한 원칙: 버킷 전체가 아닌 특정 경로로 제한을 권장한다.
> ```json
> "Resource": "arn:aws:s3:::my-iot-data-bucket/telemetry/*"
> ```

### 4.2 Role 연결 확인

```bash
# Core Device의 IoT Thing과 연결된 Role Alias 확인
aws iot describe-thing --thing-name MyCoreThing \
  | jq '.thingArn'

# Greengrass 토큰 교환 서비스 Role 확인
aws greengrassv2 get-core-device \
  --core-device-thing-name MyCoreThing \
  | jq '.status'
```

## 5. 스트림 설정 파라미터 가이드

### 5.1 `strategy_on_full` 선택

| 전략 | 설명 | 권장 사용 |
|------|------|-----------|
| `OverwriteOldestData` | 꽉 차면 가장 오래된 메시지 삭제 | 실시간 센서 (최신 데이터 우선) |
| `RejectNewData` | 꽉 차면 새 메시지 거부 (오류 반환) | 알람/이벤트 (데이터 손실 불가) |

### 5.2 `max_size` 권장값

Stream Manager 버퍼 크기는 Core Device 저장 공간과 합산하여 결정한다.

| 사용 시나리오 | 권장 `max_size` | 비고 |
|-------------|----------------|------|
| 소형 파일 고빈도 | 64 MB | 센서 이벤트 |
| 중형 파일 | 128 MB | 압축 텔레메트리 |
| 대형 파일 저빈도 | 256 MB | 이미지, 짧은 영상 |

> Stream Manager `max_size` + 스풀 한도(`max_spool_size_mb`)의 합이 가용 저장공간을 초과하지 않도록 한다. 자세한 계산은 [research/03](research/03-spool-sizing-rationale.md)를 참고한다.

### 5.3 `batch_size` 와 `batch_interval_millis`

여러 작은 파일을 하나로 묶어 S3 PUT 횟수를 줄일 수 있다.  
단, 배치가 채워질 때까지 S3 업로드가 지연된다.

```python
S3ExportTaskExecutorConfig(
    batch_size=10,                   # 10개 메시지가 쌓이면 업로드
    batch_interval_millis=5000,      # 또는 5초가 지나면 업로드 (먼저 도달하는 조건)
)
```
