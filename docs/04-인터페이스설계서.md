## 1. 클라이언트 인터페이스 — 스풀 파일 인코딩

클라이언트 프로세스는 별도의 IPC 없이 **파일 시스템**을 통해 스풀러와 통신한다.

### 1.1 파일명 형식

```
{stream_id}__{s3_key_encoded}
```

#### 필드 상세

**stream_id** (stream_name과 동일한 명명 규칙)
- 형식: `^[a-z][a-z0-9\-_]*$` (regex)
- 길이: 1~64자
- 구성: 소문자 영문자로 시작하고, 소문자·숫자·하이픈·언더스코어만 포함
- 금지: 대문자, `__`, 공백, 특수문자, 숫자 시작
- 권장: kebab-case (예: `telemetry`, `device-001`, `camera-front`)
- 형식 위반 시: `ValueError` 발생

**s3_key_encoded**
- 형식: S3 키 경로의 슬래시(/)를 느낌표(!)로 치환
- 원본: S3 버킷 내 전체 키 경로 (예: `data/device-1/reading.json`)
- 인코딩 예시: `data!device-1!reading.json`
- 목적: 파일시스템에서 경로 구분자 충돌 방지, 가독성 및 디버깅 편의성 제공

#### Python 인코딩 유틸리티

```python
from spooler.filename_codec import encode, decode

# 인코딩 (클라이언트가 사용)
spool_name = encode(
    stream_id="telemetry",
    s3_key="data/device-1/reading.json",
)
# 결과: "telemetry__data!device-1!reading.json"

# 스풀 디렉토리에 파일 생성
import shutil
shutil.copy("/path/to/reading.json", f"/var/spool/s3/{spool_name}")

# 디코딩 (Spooler 내부에서 사용)
meta = decode(spool_name)
# meta.stream_id   → "telemetry"
# meta.s3_key      → "data/device-1/reading.json"
# meta.original_name → "reading.json"
```

---

### 1.2 파일 쓰기 권장 순서

경쟁 조건을 방지하기 위해 다음 순서를 권장한다:

```
1. 임시 디렉토리(/tmp 등)에 최종 파일 내용 완성
2. encode()로 스풀 파일명 생성
3. atomic rename(os.rename)으로 스풀 디렉토리에 이동
   - rename은 같은 파일시스템 내에서 원자적으로 동작
   - 다른 파일시스템이면 copy + rename 사용
```

**주의**: 스풀 디렉토리에 직접 쓰면 쓰기 중인 파일이 감지될 수 있다. 반드시 atomic rename을 사용한다.

## 2. Greengrass 컴포넌트 설정 인터페이스

`recipe.yaml`의 `ComponentConfiguration.DefaultConfiguration`으로 제공되며, Greengrass 배포 시 오버라이드 가능하다.

| 설정 키 | 타입 | 기본값 | 설명 |
|---------|------|--------|------|
| `spool_dir` | string | `/tmp/s3-spooler/spool` | 스풀 디렉토리 경로 |
| `max_spool_size_mb` | int | **900** | 최대 스풀 크기 (MB) — 전용 1GB 파티션 기준 |
| `file_retention_hours` | int | `24` | 파일 보존 기간 (시간) |
| `poll_interval_seconds` | int | `5` | 정리 사이클 주기 (초) |
| `stream_manager_host` | string | `localhost` | Stream Manager 호스트 |
| `stream_manager_port` | int | `8088` | Stream Manager 포트 |
| `log_level` | string | `INFO` | 로그 레벨 |
| `registry_path` | string | `{spool_dir}/../registry.json` | 특별한 S3 버킷 라우팅용 선택적 레지스트리 파일 경로 (신규) |
| `status_stream_name` | string | `""` | A-07 업로드 완료 감지용 상태 스트림 이름. 빈 문자열이면 미사용 (신규) |

## 3. Stream Manager 인터페이스

Spooler는 Stream Manager의 기존 S3 Export 스트림에 메시지를 append한다.  
스트림 생성/설정은 이 컴포넌트의 책임 범위가 아니다.

### 3.1 전제 조건

- `stream_id`에 해당하는 스트림이 Stream Manager에 미리 생성되어 있어야 한다
- 스트림 타입: `S3ExportTaskExecutorConfig`
- S3 버킷 및 IAM 권한은 별도 설정 필요

### 3.2 메시지 구조

```
append_message(
    stream_name=stream_id,    # 스트림 이름
    data=file_bytes,          # 파일 전체 바이트
)
```

**참고**: Stream Manager는 `data`를 S3에 저장할 때 스트림 설정의 `keyPrefix`와 조합하여 S3 키를 결정한다. `s3_key`와의 정확한 매핑은 스트림 설정에 의존한다.

## 4. 로그 인터페이스

Python `logging` 모듈을 사용하며, Greengrass의 로그 수집 컴포넌트로 전달된다.

### 로그 형식

```
{timestamp} {level} {logger_name} — {message}
```

### 주요 로그 이벤트

| 레벨 | 이벤트 | 메시지 예시 |
|------|--------|-------------|
| INFO | 전송 완료 | `전송 및 삭제 완료: stream=telemetry key=data/f.json` |
| INFO | 기동 | `스풀 디렉토리 감시 시작: /var/spool/s3` |
| WARNING | 보존기간 삭제 | `보존기간 초과로 삭제: telemetry__...` |
| WARNING | 용량초과 삭제 | `용량 초과로 삭제: telemetry__... (614400 bytes)` |
| ERROR | 전송 실패 | `전송 실패: ... — Connection refused (재시도 대기)` |
| ERROR | 파일 삭제 실패 | `파일 삭제 실패: /path/... — Permission denied` |
