# Deployment Guide — GGC S3 Spooler

## 개정이력

| 버전 | 날짜 | 작성자 | 내용 |
|------|------|--------|------|
| 1.0.0 | 2026-05-31 | 초안 | 최초 작성 |

## 1. 개요

이 문서는 GGC S3 Spooler를 Greengrass Core Device에 배포하는 전체 절차를 설명한다.

### 배포 방식

| 방식 | 사용 시점 | 명령 |
|------|-----------|------|
| **수동 로컬** | 개발 중 빠른 반복 테스트 | `make deploy-local` (디바이스에서) |
| **수동 AWS** | 스테이징/운영 단계적 배포 | `make deploy-aws` / `make deploy-full` |
| **자동 AWS** | main 브랜치 릴리즈 | git tag push → GitHub Actions 자동 실행 |

## 2. 사전 준비

### 2.1 개발 머신 요구사항

```bash
# 의존성 설치
pip install -e ".[dev]"   # boto3, pyyaml 포함

# AWS 자격증명 설정 (로컬 배포 스크립트용)
aws configure             # 또는 환경변수 AWS_ACCESS_KEY_ID 등 설정
```

### 2.2 AWS 리소스 준비

| 리소스 | 목적 | 확인 명령 |
|--------|------|-----------|
| S3 버킷 | 아티팩트 저장 | `aws s3 ls s3://{BUCKET}` |
| Greengrass Stream Manager 스트림 | 실제 S3 전송 | AWS IoT 콘솔 확인 |
| IAM Role (OIDC) | GitHub Actions 자격증명 | AWS IAM 콘솔 확인 |
| Thing / ThingGroup | 배포 대상 | `aws iot list-things` |

### 2.3 배포 설정 파일 작성

```bash
# 템플릿 복사 후 실제 값 입력
cp deploy.yaml.example deploy.yaml
vi deploy.yaml   # REPLACE_WITH_* 값 교체
```

`deploy.yaml` 필수 항목:

```yaml
component:
  name: com.example.S3Spooler   # 컴포넌트 식별자 (변경 시 별개 컴포넌트)
  s3_bucket: "my-bucket"        # 아티팩트 저장 버킷
  s3_prefix: "artifacts"
  region: "ap-northeast-2"

deployment:
  target_arn: "arn:aws:iot:ap-northeast-2:123456789012:thinggroup/MyGroup"
```

## 3. 버전 관리

### 3.1 버전 체계

`MAJOR.MINOR.PATCH` Semantic Versioning 을 따른다.

| 변경 유형 | 버전 증가 | 예시 |
|-----------|-----------|------|
| 호환성 깨지는 변경 | MAJOR | 파일명 인코딩 규칙 변경 |
| 기능 추가 | MINOR | 재시도 큐 추가 |
| 버그 수정 | PATCH | 클리너 경쟁 조건 수정 |

### 3.2 버전 범프 명령

```bash
make bump-patch   # 1.0.0 → 1.0.1
make bump-minor   # 1.0.0 → 1.1.0
make bump-major   # 1.0.0 → 2.0.0

# 또는 특정 버전으로
python scripts/bump_version.py 1.2.3
```

`src/spooler/__init__.py` 와 `pyproject.toml` 이 동시에 업데이트된다.

### 3.3 릴리즈 절차 (수동)

```bash
# 1. 버전 범프 + 빌드 (한 번에)
make release VERSION=1.1.0

# 2. git commit + tag
git add src/spooler/__init__.py pyproject.toml
git commit -m "chore: bump version to 1.1.0"
git tag v1.1.0
git push origin main v1.1.0
# → GitHub Actions release.yml 자동 실행
```

## 4. 빌드

```bash
make build
```

산출물은 `dist/` 에 생성된다:

```
dist/
├── artifacts/
│   └── com.example.S3Spooler/
│       └── 1.0.0/
│           └── ggc-s3-spooler-1.0.0.zip    ← Greengrass 아티팩트
├── recipes/
│   └── com.example.S3Spooler-1.0.0.yaml    ← 최종 레시피
└── build-manifest-1.0.0.json               ← 빌드 메타데이터 (버전, SHA256 등)
```

### 아티팩트 구조

```
ggc-s3-spooler-1.0.0.zip
└── spooler/
    ├── __init__.py
    ├── __main__.py
    ├── config.py
    ├── filename_codec.py
    ├── watcher.py
    ├── cleaner.py
    ├── stream_client.py
    └── stream_registry.py
```

Greengrass가 zip을 다운로드·해제하면 `{artifacts:path}/spooler/` 가 생성된다.  
레시피의 Run step에서 `PYTHONPATH={artifacts:path}` 로 패키지를 인식한다.

## 5. 수동 배포

### 5.1 AWS 배포 (S3 + 컴포넌트 등록)

```bash
# 컴포넌트 등록만 (디바이스 배포 없음)
make deploy-aws

# 컴포넌트 등록 + 디바이스 즉시 배포
make deploy-full

# 배포 계획만 확인 (AWS 호출 없음)
make deploy-dry
```

내부 실행 순서:
1. `dist/` 의 빌드 매니페스트 탐색
2. zip을 `s3://{bucket}/artifacts/{ComponentName}/{version}/` 에 업로드
3. `greengrassv2:create_component_version` 호출 (레시피 base64 인코딩 전달)
4. (`--deploy` 시) `greengrassv2:create_deployment` 호출

### 5.2 로컬 Core Device 배포

이 스크립트는 **Core Device에서 직접 실행**한다. S3 없이 로컬 파일을 사용한다.

```bash
# 개발 머신에서 dist/ 를 Core Device로 전송
rsync -av dist/ user@core-device:/home/user/s3-spooler/dist/
scp -r dist/ user@core-device:~/s3-spooler/

# Core Device 에서 실행
cd ~/s3-spooler
bash scripts/deploy_local.sh          # 최신 빌드 배포
bash scripts/deploy_local.sh 1.0.0    # 특정 버전 배포
bash scripts/deploy_local.sh --status # 배포 상태 확인
bash scripts/deploy_local.sh --remove # 컴포넌트 제거
```

## 6. 자동 배포 (GitHub Actions)

### 6.1 워크플로우 구성

| 워크플로우 | 트리거 | 동작 |
|------------|--------|------|
| `ci.yml` | PR/push to main | 린트 + 타입체크 + 테스트 + 빌드 dry-run |
| `release.yml` | `v*.*.*` 태그 push | 테스트 검증 → 빌드 → S3 업로드 → 컴포넌트 등록 → 디바이스 배포 → GitHub Release |
| `release.yml` | workflow_dispatch | 수동 버전 지정 배포 |

### 6.2 GitHub Secrets 설정

GitHub 저장소 → Settings → Secrets and variables → Actions:

| Secret 이름 | 값 | 설명 |
|-------------|-----|------|
| `AWS_DEPLOY_ROLE_ARN` | `arn:aws:iam::123456789012:role/GithubActionsDeployRole` | OIDC 인증용 IAM Role |
| `AWS_REGION` | `ap-northeast-2` | AWS 리전 |
| `GGC_ARTIFACT_S3_BUCKET` | `my-greengrass-artifacts` | 아티팩트 S3 버킷 |
| `GGC_DEPLOYMENT_TARGET_ARN` | `arn:aws:iot:...:thinggroup/MyGroup` | 배포 대상 ARN |

### 6.3 OIDC IAM Role 정책 (최소 권한)

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:PutObject", "s3:GetObject"],
      "Resource": "arn:aws:s3:::my-greengrass-artifacts/artifacts/*"
    },
    {
      "Effect": "Allow",
      "Action": [
        "greengrassv2:CreateComponentVersion",
        "greengrassv2:CreateDeployment"
      ],
      "Resource": "*"
    }
  ]
}
```

### 6.4 수동 트리거 (workflow_dispatch)

GitHub Actions 탭 → Release → Run workflow:
- `version`: 배포 버전 (비워두면 최신 빌드)
- `deploy_to_devices`: 체크 시 디바이스에 즉시 배포

### 6.5 배포 보호 (Environment Protection)

`production` Environment를 GitHub에 생성하고 Reviewers를 지정하면  
자동 배포 전 수동 승인이 필요하게 된다.

## 7. 배포 후 검증

```bash
# Core Device에서 컴포넌트 상태 확인
sudo /greengrass/v2/bin/greengrass-cli component list

# 컴포넌트 로그 확인
sudo tail -f /greengrass/v2/logs/com.example.S3Spooler.log

# 스풀 디렉토리 테스트 (배포 후)
python -c "
from spooler.filename_codec import encode
import shutil, pathlib
spool = pathlib.Path('/var/spool/s3-spooler/spool')
name = encode('my-stream', 'test/hello.txt')
(spool / name).write_bytes(b'hello greengrass')
print('파일 생성:', spool / name)
"
# → 로그에서 전송 완료 확인
```

## 8. 롤백

AWS 배포는 이전 컴포넌트 버전으로 롤백할 수 있다:

```bash
# AWS CLI로 이전 버전 배포
python scripts/deploy_aws.py --version 1.0.0 --deploy

# 또는 AWS IoT 콘솔 → Greengrass → Deployments → 이전 배포 재활성화
```
